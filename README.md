# Анализ тональности отзывов с RuBERT

Проект реализует сервис анализа тональности русскоязычных отзывов на базе `DeepPavlov/rubert-base-cased`, `PyTorch`, `Transformers` и `FastAPI`. Решение подготовлено под развертывание в `VK Cloud` с хранением моделей в `VK Object Storage`.

## Что входит в проект

- дообучение `RuBERT` на задаче классификации тональности;
- предобработка датасета `RuSentiment` с приведением к 3 классам: `positive`, `neutral`, `negative`;
- HTTP API для инференса новых отзывов;
- контейнеризация через Docker;
- подготовка к деплою на `Ubuntu 22.04` в `VK Cloud`;
- опциональная загрузка и выгрузка модели в `VK Object Storage`.

## Структура проекта

- `src/data/` — загрузка и предобработка датасета;
- `src/services/` — FastAPI сервис;
- `src/storage/` — работа с `VK Object Storage` через S3 API;
- `src/train.py` — обучение модели;
- `configs/config.yaml` — основные настройки проекта;
- `scripts/bootstrap_ubuntu.sh` — настройка Ubuntu 22.04 на ВМ;
- `scripts/deploy_to_vm.sh` — деплой контейнера на ВМ;
- `docker-compose.yml` — запуск сервиса в контейнере;
- `.env.example` — пример переменных окружения.

## Подготовка окружения

1. Установите зависимости:

```bash
pip install -r requirements.txt
```

2. Подготовьте датасет:

- поместите файл `train.csv` в `data/raw/train.csv`;
- обязательные колонки: `text`, `label` или `sentiment`;
- для `RuSentiment` будут автоматически оставлены только классы `positive`, `negative`, `neutral`.

3. При необходимости создайте файл `.env` на основе `.env.example`.

## Обучение модели

```bash
python -m src.train
```

После обучения модель будет сохранена в `models/rubert_sentiment_model`.

Если заданы переменные `S3_ACCESS_KEY`, `S3_SECRET_KEY`, `S3_BUCKET` и `S3_MODEL_PREFIX`, модель дополнительно загрузится в `VK Object Storage`.

## Локальный запуск API

```bash
uvicorn src.services.api:app --host 0.0.0.0 --port 8000 --reload
```

Проверка:

```bash
curl http://127.0.0.1:8000/health
```

Веб-интерфейс доступен в браузере по адресу:

```text
http://127.0.0.1:8000/
```

На главной странице можно ввести текст отзыва и получить результат классификации (тональность и уверенность модели). Также поддерживается отправка по `Ctrl+Enter`.

Пример запроса через curl:

```bash
curl -X POST http://127.0.0.1:8000/predict \
  -H "Content-Type: application/json" \
  -d '{"text":"Сервис отличный, все понравилось"}'
```

## Рекомендуемое окружение деплоя в VK Cloud

Для дешевого и достаточного деплоя API рекомендована отдельная ВМ без GPU:

- ОС: `Ubuntu 22.04`;
- конфигурация: `STD2-2-4`;
- диск: `SSD 30 ГБ`;
- сеть: внешний IP;
- группы безопасности: `ssh` и `www` либо `ssh+www`.

Почему именно так:

- `RuBERT` для инференса можно запускать на CPU;
- GPU в `VK Cloud` требует отдельного подключения квоты;
- самый дешевый подходящий GPU-флейвор стоит существенно дороже обычной ВМ.

Ориентировочная стоимость деплоя API на 30 дней по прайс-листу VK Cloud от 12.01.2026:

- `2 vCPU Intel Cascade Lake`: `2 x 820 = 1640 ₽`;
- `4 ГБ RAM`: `4 x 223 = 892 ₽`;
- `SSD 30 ГБ`: `30 x 13 = 390 ₽`;
- `плавающий IPv4`: `188,39 ₽`.

Итого: около `3110 ₽/месяц`.

## Стоимость GPU для обучения

Если обучение нужно провести именно в `VK Cloud`, минимальный подходящий GPU-флейвор из прайс-листа — `GPU2-16-72-L4-1`:

- `16 CPU`, `72 ГБ RAM`, `24 ГБ VRAM`;
- `71 456 ₽/30 дней`.

Важно: GPU-флейворы в `VK Cloud` подключаются по заявке через поддержку, они не доступны в проекте автоматически.

## Развертывание на ВМ VK Cloud

1. Создайте ВМ `Ubuntu 22.04` в `VK Cloud`.
2. Подключитесь по SSH.
3. Выполните на ВМ:

```bash
chmod +x scripts/bootstrap_ubuntu.sh
./scripts/bootstrap_ubuntu.sh
```

4. С локальной машины передайте проект и выполните деплой:

```bash
chmod +x scripts/deploy_to_vm.sh
./scripts/deploy_to_vm.sh ubuntu@<IP_ВМ> ~/.ssh/<ключ>.pem
```

## VK Object Storage

```text
https://hb.ru-msk.vkcloud-storage.ru
```

Регион для `AWS CLI`:

```text
ru-msk
```

## Замечания

- если локальная модель отсутствует, API может скачать ее из `VK Object Storage` при наличии нужных переменных окружения;
- в проекте оставлен демонстрационный режим с генерацией маленького тестового датасета, если `train.csv` отсутствует;
- для учебного проекта лучше разделять окружение обучения и окружение продакшен-деплоя API.
